## ğŸ¯ CORE REQUIREMENTS

### Functional Requirements

#### 1. MCIA Navigator App (Passenger Interface)
- [ ] Flight number input & automatic data retrieval
- [ ] Real-time travel timeline generation
- [ ] Transportation mode selection with ETA recalculation
- [ ] Drop-off zone optimization recommendations
- [ ] Pre-arrival dining/lounge booking suggestions
- [ ] Live queue wait time displays (Check-in, Security, Immigration)
- [ ] AR-based wayfinding (optional enhancement)
- [ ] Restaurant recommendations with walk times
- [ ] Boarding alerts with dynamic gate updates
- [ ] Delay notifications with adjusted timelines
- [ ] Gate change instant alerts with re-routing
- [ ] Multi-language support (EN, Tagalog, Cebuano minimum)

#### 2. AOCC Predictive Dashboard (Operator Interface)
- [ ] Live airport heatmap with occupancy analytics
- [ ] 30-60 minute predictive queue forecasting
- [ ] Color-coded alert system (Green/Yellow/Orange/Red)
- [ ] Staff deployment recommendations
- [ ] One-tap action acceptance for AI suggestions
- [ ] Auto-logging for shift reports
- [ ] Historical data visualization
- [ ] Disaster protocol activation
- [ ] Real-time camera feed integration display
- [ ] Multi-zone monitoring (Domestic/International/Gates)

### Non-Functional Requirements

#### Performance
- App load time: < 2 seconds
- Real-time data refresh: Every 30 seconds
- Push notification delivery: < 5 seconds latency
- Dashboard data refresh: Every 15 seconds
- AR navigation frame rate: Minimum 30fps

#### Security
- Secure API authentication (OAuth 2.0)
- Encrypted data transmission (TLS 1.3)
- Role-based access control (RBAC)
- Passenger data anonymization
- GDPR/Privacy compliance

#### Scalability
- Support 10,000+ concurrent passenger users
- Handle 50+ simultaneous operator sessions
- Vertex AI API rate limit management
- Graceful degradation if AI service unavailable

## ğŸš« DESIGN CONSTRAINTS & RULES

### What AI Must ALWAYS Remember

#### General Rules
1. **Mobile-first design** - Design for smallest screen first
2. **One primary action per screen** - Avoid decision paralysis
3. **Accessibility is mandatory** - WCAG 2.1 AA minimum
4. **Performance over features** - Fast > Feature-rich
5. **User ignorance assumption** - No assumption of prior knowledge
6. **Consistent naming** - Same term everywhere (e.g., "Gate" not "Boarding Gate" sometimes)

#### Passenger App Rules
1. **Maximum 3 taps to any feature** - Deep navigation = lost users
2. **Critical info above the fold** - Flight status, queue times always visible
3. **No login required for basic features** - Reduce friction
4. **Green = good, red = urgent** - Universal color meaning
5. **Font size minimum 16px** - Stressed users need readability
6. **Avoid jargon** - "Security checkpoint" not "STCP"
7. **Always show time remaining** - "15 minutes until boarding" not "Boards at 3:45pm"
8. **Location permission must be optional** - Core features work without

#### Operator Dashboard Rules
1. **Alerts must be dismissible** - Avoid notification fatigue
2. **Actions require confirmation** - Prevent accidental deployment
3. **Use progressive disclosure** - Hide complexity until needed
4. **Numbers must use monospace font** - Easier to scan
5. **Red alerts blink slowly (1s interval)** - Urgent but not seizure-inducing
6. **Historical data uses relative time** - "20 min ago" not "14:32"
7. **Predictions show confidence level** - "85% confident" transparency
8. **Every auto-action must be reviewable** - Audit trail mandatory

### What AI Must NEVER Do

#### Passenger App Prohibitions
- âŒ **Never auto-play videos/sounds** - Startling in public spaces
- âŒ **Never require account creation upfront** - Friction kills adoption
- âŒ **Never show more than 5 items in a list** - Use "Show more" instead
- âŒ **Never use carousel for critical info** - Users don't swipe
- âŒ **Never hide navigation in hamburger menu** - Make navigation obvious
- âŒ **Never use ambiguous icons without labels** - "What does this mean?"
- âŒ **Never display technical errors** - Show "Try again" instead
- âŒ **Never use pure red/green for status alone** - Color blind users exist

#### Operator Dashboard Prohibitions
- âŒ **Never auto-dismiss critical alerts** - Operator must acknowledge
- âŒ **Never show predictions without data source** - Transparency required
- âŒ **Never use light text on light bg** - Dark mode contrast rules
- âŒ **Never block urgent alerts with modals** - Always accessible
- âŒ **Never show raw API errors** - Log them, show user-friendly message
- âŒ **Never use animation > 0.5s** - Operators need speed
- âŒ **Never require horizontal scrolling** - All info should fit viewport
- âŒ **Never use auto-refresh without visual indicator** - Operators need to know when data changes

### Frontend Stack (Recommended)
```
Framework: React (Web) / React Native (Mobile)
Styling: Tailwind CSS + CSS Modules
State Management: Zustand or React Context
Routing: React Router
Data Fetching: TanStack Query (React Query)
Maps: Google Maps API / Mapbox
Charts: Recharts or Chart.js
Icons: Lucide React
Animations: Framer Motion
```

### Backend Integration Points
```
1. Flight Data: AIDX (Aviation Info Data Exchange)
2. AI Processing: Google Vertex AI Studio
3. Computer Vision: Vertex AI Vision (occupancy analytics)
4. Traffic Data: External APIs (Google Maps, Waze)
5. Weather: OpenWeather or PAGASA API
6. Internal: MCIA AOCC Database
7. Notifications: Firebase Cloud Messaging (FCM)
```

### API Response Time SLAs
```
Critical (Flight Status, Alerts):     < 1 second
High Priority (Queue Times):          < 2 seconds
Standard (Recommendations):           < 3 seconds
Low Priority (Historical Data):       < 5 seconds
```

## ğŸ“ DESIGN PRINCIPLES SUMMARY

### For Passenger Experience
1. **Anticipate Anxiety** - Answer questions before they're asked
2. **Progressive Disclosure** - Show only what's needed now
3. **Consistency is Comfort** - Same patterns throughout
4. **Speed Over Beauty** - Fast loading > fancy animations
5. **Forgiveness Over Prevention** - Easy undo, hard to break

### For Operator Experience
1. **Glanceable Intelligence** - Understand status in 5 seconds
2. **Action Over Analysis** - Recommend solutions, not just data
3. **Trust Through Transparency** - Show AI reasoning
4. **Calm Technology** - Alert only when necessary
5. **Audit Everything** - Every action logged and reviewable
